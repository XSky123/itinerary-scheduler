# 行程安排工具

> 这是一个由 **Claude Code 4.6 + Codex 5.6 sol** 纯 vibe coding 完成的试水项目，存在各种奇怪问题的可能。请善用 [Issues](https://github.com/xsky123/itinerary-scheduler/issues) 功能反馈问题。

一个纯前端、约束驱动的可视化行程排程工具。录入一天内的交通班次，在甘特图上调整时间并组合多个计划，应用会实时检查时间冲突并生成可编辑的行程预览。

[在线使用](https://xsky123.github.io/itinerary-scheduler/) · [📖 看图就会的超简单教程](docs/tutorial.md)

## 功能

- 按交通行管理航班、铁路、巴士、接驳和自定义班次。
- 新用户首次打开自带根室官方时刻快照：去返 4 条交通行、20 个候选班次，覆盖清晨至夜间。
- 可粘贴多行文本批量导入整张时刻表。
- 单击候选班次即可更新当前激活计划；同一交通行的新班次会自动替换旧班次。
- 单击任意计划行可切换当前计划；右键菜单仍可精确加入或从指定计划移除。
- 在甘特图中拖动或缩放班次，按 5 分钟吸附。
- 双击班次块编辑，拖拽超过移动阈值时只调整时间，不会触发点击选择。
- 自动按出发时间排序；班次直接重叠时会把连接明确标红。
- 在计划间隙中添加、拖动和缩放自定义事项。
- 候选班次允许重叠并自动分轨，保持备选方案可读。
- 计划事项不会与班次或其他事项重叠；冲突修改会被拒绝并显示原因。
- 支持撤销/恢复、可撤销的一键清空、键盘移动、响应式窄屏布局和精美 HTML 行程单导出；HTML 可直接打印为 PDF。
- “一键清空”和“恢复官方示例”都需二次确认，且都可撤销。
- 行程预览与导出文件按“一段交通一行”排版，起止时间同行显示。
- 数据仅保存在浏览器 `localStorage`，不上传服务器。

## 拖拽和重叠规则

| 场景 | 行为 |
| --- | --- |
| 班次库中的候选班次重叠 | 允许；同一交通行自动纵向分轨。 |
| 同一计划内的班次直接重叠 | 允许形成备选方案，但连接会被标记为无效。 |
| 班次不重叠但少于建议缓冲 | 当前保持可行；缓冲值用于排程信息，不作为硬阻断。 |
| 事项拖向相邻班次或事项 | 自动限制在当前可用间隙内，不允许覆盖。 |
| 修改班次后会覆盖计划事项 | 拒绝修改并恢复原位置，同时显示冲突原因。 |

## 默认时刻数据来源

- [JR 北海道：根室本线（钏路-根室）下行时刻表](https://jrhokkaidonorikae.com/vtime/vtime.php?s=2930) 与 [上行时刻表](https://jrhokkaidonorikae.com/vtime/vtime.php?s=2931)，页面注明数据以 2026 年 7 月号 JR 时刻表为基准。
- [根室交通：纳沙布线官方 PDF](https://www.nemurokotsu.com/manage/wp-content/uploads/2026/03/b89fa631ac14e1911f3a83e1ad37b057.pdf)，表内注明令和 3 年 10 月 1 日改定。

时刻快照只用于展示排程逻辑。临时运休、运转日和后续改点请以运营方出发前公告为准。

## 本地开发

要求 Node.js 20.19+，或 Node.js 22.12+。

```bash
npm ci
npm run dev
```

常用命令：

```bash
npm run type-check   # TypeScript 检查
npm run build        # 类型检查并生成单文件 dist/index.html
npm run check        # 提交前完整检查
npm run preview      # 预览生产构建
```

## 技术栈

- React 18、TypeScript、Zustand、dayjs
- Vite 与 `vite-plugin-singlefile`
- GitHub Actions + GitHub Pages

## 数据与隐私

所有班次、计划、事项和设置都存放在当前浏览器的 `itinerary-scheduler-v1` localStorage 项中。清理站点数据会删除本地计划。
